SCAN FAITH

NEXT-GEN DIETARY SCANNER

THE PROJECT

"Eat Pure. Live Pure."

ScanFaith is a web-based utility that helps users instantly identify Halal, Haram, or Doubtful products. It runs directly in the browser—no App Store download required—utilizing a high-performance Python backend and a custom "Glassmorphism" UI for a premium user experience.

THE OBJECTIVE

To build a privacy-first, intelligent analysis tool that automates dietary verification. The goal was to interface with the Open Food Facts API to flag specific non-halal additives (like E-codes, Gelatin, and Ethanol) in real-time.

THE TECH STACK

  • Python (Flask) Backend
  • Open Food Facts API (Data Source)
  • GSAP (Fluid Animations)
  • HTML5-Qrcode (Camera Logic)

KEY LEARNINGS

This project deepened my understanding of API integration and data filtering. I learned how to build custom algorithms to parse complex ingredient lists against a curated database of "Haram" keywords while maintaining privacy by keeping history local.

CHALLENGES & SOLUTIONS

The Challenge: Mobile Browser Security. Modern mobile browsers block camera access on insecure (HTTP) connections, making local testing difficult.

The Solution: I learned to deploy the app to secure environments (like Render) and used ngrok to create HTTPS tunnels during development, ensuring the barcode scanner worked seamlessly on actual mobile devices.
